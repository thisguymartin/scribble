# AGENTS.md

## Communication

- Direct, casual, first-person tone.
- Keep updates short and terminal-first.
- Use `->` notation for flows.
- Include sources and confidence levels for factual guidance.

## Skill Authoring

- Keep `SKILL.md` files concise.
- Use lowercase hyphen names.
- Put reusable details in references only when a skill grows too large.
- Do not put README-style docs inside individual skill folders.

## Privacy

- Do not send real customer data to third-party AI services.
- Use synthetic data in examples and tests.
- Prefer local codebase, git, and provided files before external sources.

## Commits

- Use conventional commit subjects under 72 chars.
- Include a detailed body when multiple files or concerns changed.
- Never include AI attribution, co-author trailers, or signed-off-by lines.
