One-time builder for the /deploy-html skill. Fill in the inputs on the left; the commands and configs on the right update live and copy in one click.

How to use: set worker name and account subdomain (from your Cloudflare dashboard URL), click Generate token, then walk through the 8 setup steps top-to-bottom. The final block is the ~/.scribble-deploy/config.json you save to your home directory.

Inputs

becomes <name>.<account>.workers.dev
find it at dash.cloudflare.com → Workers & Pages → your subdomain
paste the same value into wrangler secret put UPLOAD_TOKEN

Architecture

Setup steps

1 Sign up for Cloudflare (free tier is enough) browser 
open https://dash.cloudflare.com
2 Install Wrangler & log in terminal 
# in the scribble repo root
cd worker
npm install
npx wrangler login
3 Create the D1 database copies database_id you paste back into wrangler.toml 
npx wrangler d1 create scribble-artifacts
4 Apply migrations creates the artifacts table 
npx wrangler d1 migrations apply scribble-artifacts --remote
5 Set the UPLOAD_TOKEN secret paste the token from the left panel 
# the value you pasted: <token>
npx wrangler secret put UPLOAD_TOKEN
6 Deploy the Worker prints the live URL on success 
npx wrangler deploy
# expected output ends with:
#   Published scribble-deploy (...)
#     https://scribble-deploy.<account>.workers.dev
7 Save your local config ~/.scribble-deploy/config.json 
mkdir -p ~/.scribble-deploy
cat > ~/.scribble-deploy/config.json <<'JSON'
{ ... }
JSON
8 Smoke-test from the scribble repo should print a shareable URL 
# generate any HTML artifact first via an html-* skill, then:
/deploy-html

CREATE TABLE IF NOT EXISTS artifacts (
  id TEXT PRIMARY KEY,
  content BLOB NOT NULL,
  content_type TEXT NOT NULL DEFAULT 'text/html',
  filename TEXT,
  size_bytes INTEGER NOT NULL,
  created_at INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_artifacts_created_at ON artifacts(created_at DESC);
